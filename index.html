Welcome to d-verse

A decentralized framework for building trust-aware, capsule-powered apps that run securely on user devices — no servers, no chains.

Why d-verse?

d-verse empowers developers to create offline-first applications with verifiable data and peer-to-peer sync, redefining digital systems without cloud dependency.

Core Modules

dverse-runtime

Secure WASM runtime for executing signed logic securely on-device.

dverse-dag

Trust-aware DAG for tracking capsule lineage and validating identities.

dverse-did

Decentralized identity layer with keypair generation and signature verification.

dverse-net

P2P sync with DHT-based routing and optional relay support.

dverse-relay

Federated relay nodes for NAT traversal, moderation, and sync caching.

dverse-storage

Encrypted, hash-addressed blob storage for portable content sharing.

d-verse API in Rust

use dverse::{Runtime, NetConfig, Capsule};
use std::error::Error;

#[tokio::main]
async fn main() -> Result<(), Box> {
    // Initialize d-verse runtime
    let runtime = Runtime::new().await?;

    // Configure P2P network
    let net_config = NetConfig::default()
        .with_dht(true)
        .with_relay("relay.dverse.io:8080");
    runtime.start_network(net_config).await?;

    // Create and sign a capsule
    let mut capsule = Capsule::new("sample_data".to_string());
    capsule.sign(&runtime.keypair())?;

    // Sync capsule via P2P
    runtime.sync_capsule(&capsule).await?;

    println!("Capsule synced: {:?}", capsule.hash());
    Ok(())
}
